# Node 26 CI Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run CI, smoke, and release workflows on Node 26.5.0 while preserving Node 22 package compatibility.

**Architecture:** Keep runtime policy in existing workflow and README fields. Normalize npm's version-dependent pack JSON at the supply-chain boundary, leaving all downstream file validation unchanged.

**Tech Stack:** GitHub Actions, Node.js 26.5.0, npm 12, TypeScript, Vitest

## Global Constraints

- Keep `package.json` and `package-lock.json` engines at `>=22.19.0`.
- Keep the existing Node 22 `@types/node` baseline.
- Add no dependencies or CI matrix.

### Task 1: Accept npm 12 package metadata

**Files:**
- Modify: `scripts/supply-chain-guard.ts`
- Test: `tests/supply-chain-guard.test.ts`

**Interfaces:**
- Consumes: npm 11 `PackResult[]` or npm 12 `Record<string, PackResult>` JSON.
- Produces: `parsePackageFiles(stdout: string): string[]`.

- [ ] **Step 1: Add parser coverage for both npm output shapes**

Import `parsePackageFiles` and assert that an array and a single object containing `{ files: [{ path: "package.json" }] }` both return `["package.json"]`.

- [ ] **Step 2: Run the focused test and verify it fails**

Run: `mise exec node@26.5.0 -- npm test -- --exclude '.worktrees/**' tests/supply-chain-guard.test.ts`
Expected: FAIL because `parsePackageFiles` is not exported.

- [ ] **Step 3: Normalize the parsed value once**

Implement:

```ts
export function parsePackageFiles(stdout: string): string[] {
  const parsed = JSON.parse(stdout) as PackResult[] | Record<string, PackResult>;
  return (Array.isArray(parsed) ? parsed : Object.values(parsed)).flatMap(
    (result) => result.files?.map((file) => file.path).filter(Boolean) ?? [],
  ) as string[];
}
```

Use the helper from `listPackageFiles()`.

- [ ] **Step 4: Run the focused test and supply-chain guard**

Run: `mise exec node@26.5.0 -- npm test -- --exclude '.worktrees/**' tests/supply-chain-guard.test.ts && mise exec node@26.5.0 -- npm run supply-chain:guard`
Expected: PASS.

- [ ] **Step 5: Commit the package-output compatibility change**

```bash
git add scripts/supply-chain-guard.ts tests/supply-chain-guard.test.ts
git commit -S -m "fix: support npm 12 package metadata"
```
